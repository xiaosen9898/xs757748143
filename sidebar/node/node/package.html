NPM 包管理工具 ​

npm的使用 ​

TIP

1. 概念 ​

  • npm是node的包管理工具，它的出现是为了解决 Node中第三方包共享的问题

 

2. 安装 ​

  • npm在安装node时就自动安装了

 

3. 使用步骤 ​

  1. 打开项目的命令小黑窗；
  2. 初始化：
  1. npm init：生成了一个package.json文件
  2. npm init -y：快速创建
  1. 查找工具包：去npmjs.com网站 搜索 关键字 date 找到合适的格式化日期包；
  2. 安装工具包：npm install 包名称，（包名称@版本号 可以指定包下载）；
  3. 引入并使用：通过require引入并使用第三方包；
  4. 删除依赖：
  1. 局部删除：npm r 依赖名 or npm remove 依赖名
  2. 全局删除：npm remove -g 依赖名

 

4. 文件说明 ​

  1. node_modules，依赖包，下载的第三方模块放置的路径（注意：不要随意删除里面的文件）
  1. 生产依赖：npm i 包名 --save，开发和生产阶段都要用的
  2. 开发依赖：npm i 包名 --save-dev，开发阶段使用的依赖
  1. package.json：保存文件基本信息
  2. package-lock.json：保存模块信息，使用npm下载第三方模块时自动生成的文件

 

5. 安装类型 ​

  1. 环境：
  1. 生产依赖包：npm i --save 包名 or npm i -S 包名
  • 开发的时候需要使用，部署上线也要使用，如axios；
  • 默认的包的类型,可以省略不写，也可以简写为-S
  1. 开发依赖包：npm i --save-dev 包名 or npm i -D 包名
  • 开发的时候需要使用，部署上线不再需要使用，简写为 -D

 

  1. 范围：
  1. 局部安装：npm i 包名
  • 包安装在当前项目的根目录下的node_modules中；
  • 要通过require引入使用的就需要局部安装
  1. 全局安装：npm i 包名 -g
  • 包被安装到了全局的node_modules中；
  • 要用到该包的命令执行任务的就需要全局安装

 

  1. 指定版本：npm i 包名@版本号

 

6. npm卸载包 ​

  1. 本地包：
  1. 进入到你想要卸载的包所在的文件夹，到package.json这一层即可；
  2. 打开cmd小黑窗；
  3. 在小黑窗中执行npm uninstall 包名，简写：npm un 包名
  1. 全局包：
  1. 在任意地方打开小黑窗；
  2. 输入npm uninstall 全局包名 -g ，简写：npm un 全局包名 -g

 

7. 配置命令别名 ​

  1. 在package.json文件中配置：
js
"scripts": {
   "dev": "node ./server.js",
   "start": "node ./index.js",
},

  • 使用：npm run server，运行server.js文件
  • 使用：npm run start，运行index.js文件， 使用‘start’可以省略‘run’：npm start

 

cnpm ​

TIP

  1. 概念
  • cnpm 是一个淘宝构建的 npmjs.com 的完整镜像

  2. 作用

  • cnpm服务部署在国内阿里云服务器上，可以提高包的下载速度

  3. 安装

  • npm install -g cnpm --registry=https://registry.npmmirror.com

  4. 操作命令

  • 把npm换成 cnpm

 

npm 配置淘宝镜像（推荐） ​

TIP

  1. 作用：提高包的下载速度
  2. 工具配置（推荐）：
  1. 安装 nrm：npm i -g nrm；
  2. 修改镜像：nrm use taobao；
  3. 检查是否配置成功：
  1. 命令：npm config list；
  2. 出现 “The registry has been changed to 'taobao'.” 即切换成功
  1. 其他命令；
  • 查看镜像源列表：nrm ls
  • 切回官方服务源：nrm use npm，往npm官方上传包时需要切回npm

 

yarn包管理工具 ​

TIP

  1. 作用：也是一个包管理工具，速度更快
  2. 全局安装：npm i yarn -g
  3. 常用命令：
  • 初始化：yarn init or yarn init -y；
  • 安装包：yarn add 包名；
  • 删除包：yarn remove or yarn global remove；
  • 安装项目依赖：yarn；
  • 运行命令别名：yarn <别名> # 不需要添加 run；

 

nodemon ​

TIP

  1. 作用：能够使用nodemon来启动web服务器并监听文件修改了可以自动重启服务器
  2. 全局安装：npm install nodemon -g
  3. 常用命令：nodemon js文件名
  4. 常见错误：出现系统禁止执行 nodemon.ps1文件的错误
  • 常见错误：以管理员身份运行PowerShell，执行：get-ExecutionPolicy，回复Restricted，表示状态是禁止的，执行：set-ExecutionPolicy RemoteSigned，输入Y

 

nvm ​

TIP

  1. 作用：用来管理 node 版本的工具，方便切换不同版本的Node.js
  2. 下载安装：选择nvm-setup.exe下载，下载地址
  3. 常用命令：nodemon js文件名
  • nvm list：显示已安装的版本
  • nvm list available：显示所有可以下载的 Node.js 版本
  • nvm install 16.20.2：安装 16.20.2 版本的 Node.js
  • nvm install latest：安装最新版的 Node.js
  • nvm uninstall 16.20.2：删除某个版本的 Node.js
  • nvm use 16.20.2：切换 16.20.2 的 Node.js
