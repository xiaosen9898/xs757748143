1. 父传子 ​

 </div>

import ChildComponent from './ChildComponent.vue';

 name: 'ParentComponent',

2. 子传父 ​

  1. 子组件通过this.$emit('事件名', '子组件的数据')给父组件发事件并传递参数；
  2. 父组件监听子组件的事件，接收参数；
vue
<template>
  <div>
    <button @click="sendDataToParent">点击传递数据给父组件</button>  
  </div>
</template>

<script>
export default {
  name: 'ChildComponent',
  methods: {
    sendDataToParent() {                                
      // 通过 $emit 触发事件并传递数据
      this.$emit('send-data', '这是来自子组件的数据');    
    }                                                   
  }
};
</script>

组件使用 v-model ​

TIP

  1. 原理：
  1. value：父组件通过value向子组件传递初始数据；
  2. input 事件：子组件通过$emit('input', newValue)向父组件发送更新后的数据
vue
<template>
  <div>
    <h1>父组件</h1>
    <p>输入框内容：{{ parentInput }}</p>
    <!-- 使用 v-model 绑定数据到子组件 -->
    <ChildComponent v-model="parentInput" />   
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';

export default {
  name: 'ParentComponent',
  components: {
    ChildComponent
  },
  data() {
    return {
      parentInput: '默认文本'
    };
  }
};
</script>

<template>
  <div>
    <input type="text" :value="value" @input="updateValue" /> 
  </div>
</template>

<script>
export default {
  name: 'ChildComponent',
  props: {
    // 接收父组件的 v-model 数据
    value: {
      type: String,
      required: true
    }
  },
  methods: {
    updateValue(event) {                          
      // 将新数据通过 input 事件发送给父组件
      this.$emit('input', event.target.value);    
    }                                             
  }
};
</script>

2. 具名插槽 ​

  1. 需求：一个组件内有多处，需要外部传入标签，进行定制
  2. 封装：
  1. 使用<slot name=“名字”></slot>占位；
  2. 每个<slot>标签之间可以设置默认内容
  1. 使用：用标签<template #名字></template> or <template v-slot:名字>替换<slot>
vue
<!-- ChildComponent.vue -->
<template>
  <div class="child-container">
    <!-- 定义具名插槽，名称为 header -->
    <header>
      <slot name="header">默认标题</slot>   
    </header>

    <!-- 默认插槽 -->
    <main>
      <slot>默认内容</slot>
    </main>

    <!-- 定义具名插槽，名称为 footer -->
    <footer>
      <slot name="footer">默认页脚</slot>   
    </footer>
  </div>
</template>

<script>
export default {
  name: 'ChildComponent'
};
</script>

keep-alive 组件 ​

TIP

  1. 背景：对于动态组件来说，切换默认会销毁旧组件，不会缓存组件
  2. 作用：
  1. 缓存组件：keep-alive可以缓存，不销毁旧组件
  2. 缓存路由页面：<keep-alive> <router-view></router-view> </keep-alive>
  1. 使用keep-alive的组件会多两个生命周期：
  1. activated：激活，就是组件显示出来的时候
  2. deactivated：失活，就是组件切换到后台不显示的时候

 

获取DOM元素 ​

TIP

  1. 背景：每个组件都会创建自己的实例，每个实例互不影响
  2. $获取：
  1. this.$data：获取当前组件的数据
  2. this.$el：获取当前组件的节点
  1. 获取dom：ref
  1. 用在标签上，获取绑定的标签
  2. 用在组件上，获取组件实例
  1. 绑定：ref
  2. 获取：$refs
vue
<!-- ChildComponent.vue -->
<template>
  <div>子组件内容</div>
</template>

<script>
export default {
  name: 'ChildComponent',
  methods: {
    sayHello() {
      console.log('Hello from ChildComponent');
    }
  }
};
</script>

$nextTick ​

TIP

  1. 背景：Vue在数据变化后，DOM更新是异步的
  2. 作用：等DOM更新后, 才会触发执行此方法里的函数体
  3. 使用：this.nextTick(( ) => { 函数体 })

 

组件的样式冲突 ​

TIP

  1. 冲突：后面的组件样式会覆盖前面的样式
  2. 解决：在style标签上使用scoped属性，让样式只对当前组件生效
  3. 原理：
  1. 组件标签会增加一个data-v-hash属性；
  2. 样式会增加属性选择器data-v-hash
