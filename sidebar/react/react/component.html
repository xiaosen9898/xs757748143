TIP

  1. 定义：使用 ES6 的 class 创建的组件

 

  1. 约定：
  • 类组件名称首字母大写
  • 类组件应该继承React.Component父类，从而可以使用父类中提供的方法或者属性
  • 类组件必须提供render方法，并返回组件的结构，如无需要返回的结构，返回null

      </div>
    );

   }
}

  1. 关于：setState()

  1. 作用：修改组件状态，并且会数据驱动视图更新

  2. 语法1：this.setState({ key: newValue })

  • 注意：
  • react中setState是异步更新的
  • 连续调用多个setState()，会合并成一个调用
  • 要更新的数据，放到一个更新队列里面，合并执行
  • 最后一个setState()会覆盖前面所有的setState()
  1. 语法2：setState((preState) => {})
  • 注意：
  • React会把上一个setState的结果传入这个函数
  • 后面不会覆盖前面，形参代表上一次调用的state结果
  • 适用于需要调用多次setState
  1. setState()的第二个参数：
  1. 语法：(prev) => ({key: prev.key + 1}), () => {}
  2. 使用场景：在状态更新（页面完成重新渲染）后立即执行某个操作
  3. 说明：
  • 第二个参数是回调函数，会在状态更新后立即执行，相当于Vue的 $nextTick
  • 回调函数中调用setState，render会多次执行

function Child(props) {
  return (
    <div>
      <p>Message: {props.message}</p>
      <p>User: {props.user.name}</p>
    </div>
  );
}

export default Parent;

2. 子传父 ​

  1. 说明：在react中，子传父是通过事件回调来实现的
  2. 步骤：
  1. 父组件提供一个函数
  2. 父组件将该函数作为标签属性的值，传递给子组件
  3. 子组件通过 props 接收函数
  4. 子组件调用函数
  1. 使用：
js
import React, { useState } from 'react';

// ====== 父组件 ======
function Parent() {
  const [data, setData] = useState('');  // 用于接收子组件的数据

  // 1. 定义回调函数，更新父组件的数据
  const handleDataChange = (newData) => {   
    setData(newData);                       
  };                                        

  return (
    <div>
      <h1>Parent Component</h1>
      <p>Data from Child: {data}</p>
      {/* 2. 将回调函数作为props传递给子组件 */}
      <Child onDataChange={handleDataChange} />
    </div>
  );
}

// ====== 子组件 ======
function Child(props) {
  // 当输入发生变化时，调用父组件传递的回调函数
  const handleChange = (event) => {            
    // 3、4. 调用回调函数并传入数据
    props.onDataChange(event.target.value);    
  };                                           

  return (
    <div>
      <input type="text" onChange={handleChange} placeholder="Type something..." />
    </div>
  );
}

export default Parent;

3. 兄弟组件 ​

  1. 实现：将共享状态提升到最近的公共父组件中，由公共父组件管理这个状态

  2. 过程：

  1. 父组件：提供共享状态、提供操作共享状态的方法
  2. 子组件：通过 props 接收状态或操作状态的方法

 

4. 跨组件通信 ​

  1. 实现：将共享状态提升到最近的公共父组件中，由公共父组件管理这个状态

  2. 过程：

  1. 调用 React. createContext() 创建 Provider（提供数据） 和 Consumer（消费数据） 两个组件
  2. 使用 Provider 组件作为父节点（需要包裹住后代）
  3. 设置 value 属性，表示要传递的数据
  4. 调用 Consumer 组件接收数据
js
import React, { createContext, useContext, useState } from 'react';

// 1. 创建 Context
const MyContext = createContext();

function Parent() {
  const [data, setData] = useState("Hello from Context!");

  return (
    // 2. 使用 Provider 提供数据
    <MyContext.Provider value={{ data, setData }}>
      <ChildA />
    </MyContext.Provider>
  );
}

function ChildA() {
  return (
    <div>
      <h2>Child A Component</h2>
      <ChildB />
    </div>
  );
}

function ChildB() {
  const { data, setData } = useContext(MyContext);  // 3. 使用 useContext 获取数据

  return (
    <div>
      <h3>Child B Component</h3>
      <p>Received data: {data}</p>
      <button onClick={() => setData("Updated from ChildB!")}>
        Update Data
      </button>
    </div>
  );
}

export default Parent;

组件性能优化 ​

TIP

  1. 组件更新机制：组件更新，默认会导致所有后代组件更新，默认不会影响兄弟组件

  2. 减轻state：

  1. 只存储跟组件渲染相关的数据
  2. 不用做渲染的数据不要放在state中，比如定时器id等；对于需要在多个方法中用到的数据，应该直接放在this中
  1. 避免不必要的重新渲染：

  1. 背景：父组件更新会引起子组件也被更新

  2. 解决：

  1. 方式：使用钩子函数shouldComponentUpdate(nextProps, nextState)
  2. 作用：通过返回值决定该组件是否重新渲染，返回true表示重新渲染，false表示不重新渲染
  3. 触发时机：更新阶段的钩子函数，组件重新渲染前执行

 

纯组件 ​

TIP

  1. 定义：React.PureComponent与React.Component功能相似
  2. 区别：PureComponent内部自动实现了 shouldComponentUpdate钩子，不需要手动比较
  3. 原理：纯组件内部通过分别对比前后两次props和state的值，来决定是否重新渲染组件
  4. 注意：只有在性能优化的时候可能会用到纯组件，不要所有的组件都使用纯组件，因为纯组件需要消耗性能进行对比
  5. 纯组件比较
  1. 值类型：
  1. 说明：纯组件内部的对比是 shallow compare（浅层对比）
  2. 比较：对于值类型来说：比较两个值是否相同
  1. 引用类型：
  1. 说明：纯组件内部的对比是 shallow compare（浅层对比）
  2. 比较：只比较对象的引用（地址）是否相同
  • 注意：state 或 props 中属性值为引用类型时，应该创建新数据，不要直接修改原数据
js
// 正确！创建新数据
const newObj = {...state.obj, number: 2}
setState({ obj: newObj })
// 正确！创建新数据
// 不要用数组的push / unshift 等直接修改当前数组的的方法
// 而应该用 concat 或 slice 等这些返回新数组的方法
this.setState({
	list: [...this.state.list, {新数据}]
})

        </>
    )
}

  1. 常见的受控组件：文本框、文本域、下拉框、复选框

  2. 多表单元素的优化

  1. 说明：使用一个事件处理程序处理多个表单元素
  2. 步骤：
  1. 给表单元素添加name属性，名称与state属性名相同
  2. 根据表单元素类型获取对应的值
  3. 在事件处理程序中通过 [name] 修改对应的state
js
class App extends React.Component {
  state = {
    username: '',
    desc: '',
    city: "2",
    isSingle: true
  }
 
  handleChange = (e) => {
    let {name, type, value, checked} = e.target
    console.log(name, type, value, checked)
    value = type === 'checkbox' ? checked : value
    console.log(name, value)
    this.setState({
      [name]: value
    })
  }
  render() {
    return (
      <div>
        姓名：<input type="text" name="username" value={this.state.username} onChange={this.handleChange}/>
        <br/>
        描述：<textarea name="desc" value={this.state.desc} onChange={this.handleChange}></textarea>
        <br/>
        城市：<select name="city" value={this.state.city} onChange={this.handleChange}>
          <option value="1">北京</option>
          <option value="2">上海</option>
          <option value="3">广州</option>
          <option value="4">深圳</option>
        </select>
        <br/>
        是否单身：<input type="checkbox" name="isSingle" checked={this.state.isSingle} onChange={this.handleChange}/>
      </div>
    )
  }
}

非受控组件 ​

TIP

  1. 定义：表单元素的value由DOM对象控制，不受state的控制

  2. ref：值为 ref.current

  1. 作用：获取组件实例、获取DOM
  2. 步骤：
  1. 调用React.createRef()方法创建一个ref对象
  2. 将创建好的ref对象绑定到dom元素上
  3. 通过this.ref对象.current获取dom元素
js
import React, { useRef, useEffect } from 'react';

function MyComponent() {
  const myRef = useRef(null);  // 创建 ref

  useEffect(() => {
    myRef.current.focus();  // 在组件挂载后聚焦输入框
  }, []);

  return <input type="text" ref={myRef} placeholder="Focus on mount" />;   
}

export default MyComponent;

  1. 注意：
  1. 类组件可以通过ref访问子组件的实例方法
  2. 函数组件需通过forwardRef和useImperativeHandle暴露自定义方法

import React, { useRef, forwardRef, useImperativeHandle } from 'react';

// ====== 子组件 ======
const Child = forwardRef((props, ref) => {   
  // 使用 useImperativeHandle 暴露自定义方法
  useImperativeHandle(ref, () => ({          
    showAlert() {                            
      alert("Hello from Child Component!");  
    }                                        
  }));                                       

  return <div>Child Component</div>;
});

// ====== 父组件 ======
function Parent() {
  const childRef = useRef();

  const handleClick = () => {
    childRef.current.showAlert();
  };

  return (
    <div>
      <Child ref={childRef} />
      <button onClick={handleClick}>Call Child Method</button>
    </div>
  );
}

export default Parent;
