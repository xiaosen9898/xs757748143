React 路由 ​

基本使用 ​

TIP

  1. 安装：npm i react-router-dom

  2. 基本使用：

js
import React, { Suspense } from 'react'
import { Router, Route, Switch, Redirect } from 'react-router-dom'
import history from '@/utils/history'
import AuthRoute from '@/components/AuthRoute'
import KeepAlive from '@/components/KeepAlive'

const Login = React.lazy(() => import('@/pages/Login'))
const Layout = React.lazy(() => import('@/pages/Layout'))
const NotFound = React.lazy(() => import('@/pages/NotFound'))
const Article = React.lazy(() => import('@/pages/Article'))
const Chat = React.lazy(() => import('@/pages/Profile/Chat'))
const ProfileEdit = React.lazy(() => import('@/pages/Profile/Edit'))
const ProfileFeedback = React.lazy(() => import('@/pages/Profile/Feedback'))
const Search = React.lazy(() => import('@/pages/Search/index'))
const SearchResult = React.lazy(() => import('@/pages/Search/Result'))

export default function App() {
  return (
    <Router history={history}>
      <Suspense fallback={<div>loading...</div>}>
        <KeepAlive path="/layout" component={Layout} />
        <Switch>
          <Redirect from="/" to="/layout/home" exact />
          <Route path="/login" component={Login} />
          <Route path="/article/:id" component={Article} />
          <Route path="/search" component={Search} exact />
          <Route path="/search/result" component={SearchResult} />
          <AuthRoute path="/profile/edit" component={ProfileEdit} />
          <Route path="/profile/feedback" component={ProfileFeedback} />
          <AuthRoute path="/profile/chat" component={Chat} />
          <Route
            render={(_props) => {
              // 1.非/layout开头的统一404处理，/layout开的返回null不处理
              if (_props.location.pathname.startsWith('/layout')) return null
              return <NotFound />
            }}
          />
        </Switch>
      </Suspense>
    </Router>
  )
}

路由模式 ​

TIP

  1. HashRouter：
  1. 实现：使用 URL 的哈希值实现
  2. 原理：监听 window 的 hashchange`事件来实现的
  3. 重命名路由：import { HashRouter as Router, Route, Link } from 'react-router-dom'

 

  1. BrowserRouter：
  1. 实现：使用 H5 的 history API 实现
  2. 原理：监听 window 的 popstate事件来实现的
  3. 重命名路由：import { BrowserRouter as Router, Route, Link } from 'react-router-dom'

Switch ​

TIP

  1. 说明：
  1. 通常，我们会Switch组件把Route包裹着
  2. 在Switch组件中，匹配到第一个匹配的组件，停止向下匹配
  3. <Route>不写path，配合<Switch>实现404页面的提示
js
<Switch>
  <Route exact path="/" component={Home}/>
  <Route path="/about" component={About}/>
  <Route path="/user" component={User}/>
   //<Route>不写`path`，配合`<Switch>` 实现404页面的提示
  <Route component={NoMatch}/>
</Switch>

Route ​

TIP

  1. 说明：
  1. 如果 path 的路径匹配上了，那么就可以对应的组件就会被 render
  2. 如果 path 没有匹配上，那么会 render null
  3. 默认情况下，path="/" => 能够匹配任意/开始的路径，可以配合exact用以精准匹配
  4. 如果没有指定 path，那么一定会被渲染
  5. Route匹配是从上向下，依次匹配
  1. exact：
  1. exact 表示精确匹配某个路径
js
 <Route exact path="/login" component={Login}/>
1
  1. 不写path时，exact无效

 

路由导航 ​

TIP

  1. Link：
  1. Link组件最终会渲染成a标签，用于指定路由导航
  2. to属性，将来会渲染成a标签的href属性
  3. Link组件无法实现导航的高亮效果

 

  1. NavLink：
  1. 一个更特殊的Link组件，可以用用于指定当前导航高亮
  2. activeClassName，用于指定高亮的类名，默认active
  3. exact，精确匹配，表示必须精确匹配类名才生效

 

编程式导航 ​

TIP

  1. 方法：
  1. history：history 是 React 路由提供的，用于获取浏览器历史记录的相关信息，通过props.history获取
  2. push()：跳转到某个页面，参数 path 表示要跳转的路径
  3. go()：前进或后退到某个页面，参数 n 表示前进或后退页面数量（比如：-1 表示后退到上一页）
  1. 注意：
  1. 只有被<Route component={组件名}>设置过路径的组件，props上才有history
  2. HashRouter 与BroswerRouter两种模式下都具有history

嵌套路由 ​

TIP

  1. 说明：配置嵌套路由的时候，需要对路径进行处理，必须要先匹配到父级路由，才能匹配到子路由
js
// 通过/home可以匹配Home父组件  再通过/list匹配子组件
<Route path="/home/list" component={List} />
1
2
  1. 注意：
  1. 子组件中不需要再设置<Router>
  2. 父子路由，路径是可以重复的
  3. path的值，要从/一级路由路径/二级路由路径开始写
  4. 一级路由路径不要设置eaxct，会导致父子路由都无法渲染
  5. 子组件中有多个路由 ，同样推荐使用<Switch>组件包裹所有的<Router>组件

 

权限路由 ​

TIP

  1. 作用：根据用户的登录状态，动态展示不同路由

  2. 配置：

  1. 配置权限路由组件
js
import { hasToken } from '@/utils/storage/auth'
import React from 'react'
import { Redirect, Route } from 'react-router'

// 预期：
// <AuthRoute path='/profile/chat' component={ProfileChat} />

type Props = {
  path: string
  // 函数组件类型 FC：Function Component
  component: React.FC
}

export type LocationStateType = {
  fromPath: string
}

export default function AuthRoute({ path, component: Component, ...restProps }: Props) {
  return (
    <Route
      {...restProps}
      path={path}
      render={() => {
        if (hasToken()) {
          // 有 token 正常渲染组件
          return <Component />
        } else {
          // 没有 token 重定向到登录页
          return (
            <Redirect
              to={{
                pathname: '/login',
                state: { fromPath: path },
              }}
            />
          )
        }
      }}
    />
  )
}
