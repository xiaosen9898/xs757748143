NODE 模块化 ​

模块作用域 ​

TIP

每个文件就是一个模块，每个模块中的成员是私有的，只有向外暴露时，其他模块才能使用

 

CommonJS 规范 ​

TIP

  1. 概念：CommonJS，简称 CMJ，这是一个社区规范，出现时间较早，目前仅 node 环境支持；

  2. 实现：node 天生支持 CommonJS 模块化标准

  3. 规定：

  1. node 中的每个 js 文件都是一个 CMJ 模块，通过 node 命令运行的模块，叫做入口模块；
  2. 模块中的所有全局定义的变量、函数，都不会污染到其他模块；
  3. 模块可以暴露（导出）一些内容给其他模块使用，需要暴露什么内容，就在模块中给module.exports赋值；
  4. 一个模块可以导入其他模块，使用函数require("要导入的模块路径")即可完成，该函数返回目标模块的导出结果；
  5. 一个模块在被导入时会运行一次，然后它的导出结果会被 node 缓存起来，后续对该模块导入时，不会重新运行，直接使用缓存结果

注意：

  1. 导入模块时，可以省略.js；
  2. 导入模块时，必须以./或../开头

TIP

1. 暴露成员对象 ​

  • module 对象中的 exports 对象可以向外暴露自身模块的成员

3. 导入模块的基本流程 ​

  1. 将相对路径转为绝对路径，定位目标文件；
  2. 缓存检测；
  3. 读取目标文件代码；
  4. 包裹为一个函数并自执行（通过 argumentscallee.toString()查看自执行函数）；
  5. 缓存模块的值；
  6. 返回 module.exports 的值

 

4. 导入注意点 ​

  1. 导入其他类型文件时，会以 js 文件进行处理；
  2. 如果导入的路径是个文件夹：
  1. 首先会去找该文件夹下的 package.json 文件中 main 属性对应的文件，如果存在则导入，否则报错
  2. 如果 package.json 文件不存在，或者 main 属性不存在，则会尝试导入该文件夹下的 index.js 和 index.json，如果没有找到则报错

 

三大模块 ​

TIP

1. 核心模块 ​

  1. node 自带的模块，安装引入即可使用；
  2. 导入模块：const 变量名 = require('核心模块名')

2. 自定义模块 ​

  1. 自定义模块导入：const 变量名 = require('文件路径')
  1. 不能与核心模块同名；
  2. .js 后缀可以省略
  1. 自定义模块导出：通过 module.exports 关键字按需导出

3. 第三方模块 ​

  1. 定义：其他人写的模块
  2. 导入模块：const 变量名 = require('核心模块名')
