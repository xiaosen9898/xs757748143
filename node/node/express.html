Express框架 ​

TIP

  1. 定义：node中的第三方模块中的一个框架

  2. 作用：

  1. 使用express.static()托管静态资源；
  2. 使用express路由精简项目结构；
  3. 使用express中间件；
  4. 使用express创建API接口；
  5. 使用express启用cors跨域资源共享

 

使用 ​

TIP

  1. 下载安装命令：npm i express
  2. 搭建一个服务器：
js
// 引入 express 模块
const express = require('express');
const app = express();

// 设置端口
const PORT = 3000;

// 定义路由
app.get('/', (req, res) => {
  res.send('Hello, World!');
});

// 启动服务器，监听指定端口
app.listen(PORT, () => {
  console.log(`Server is running at http://localhost:${PORT}`);
});

路由 ​

TIP

  1. 概念：路由确定了应用程序如何响应客户端对特定端点的请求
  2. 组成：请求方法、请求路径、回调函数
  3. 使用：
  1. 语法：app.<method>(path，callback)
js
app.get('/', (req,res) => {
    res.send('hello');
});

  1. 其他方法：
  1. post()：用来处理请求方式为post的api
  2. all()：用来处理接口一致，请求方式不一样的api

 

  1. 获取路由参数：路由参数指的是 URL 路径中的参数（数据）
  1. 配置：app.get('/detail/:id', (req, res) => {});
  2. 获取：req.params.id

 

  1. 获取请求报文参数
  1. 兼容原生 HTTP 模块的获取方式：
  1. req.method
  2. req.url
  3. req.httpVersion
  4. req.headers
  1. express封装获取报文的方式：
  1. req.path：获取请求路径；
  2. req.query：获取查询字符串；
  3. req.get('host')：获取指定的请求头
  1. 获取请求体参数（get方式的查询字符串）：

  1. 获取请求体参数（post方式，需要使用body-parser中间件）：
  1. 下载：npm i body-parser；
  2. 使用：
js
 /* 1.导入 */
// 引入 express 和 body-parser 模块
const express = require('express');
const bodyParser = require('body-parser');

const app = express();
const PORT = 3000;

// 使用 body-parser 中间件
app.use(bodyParser.json()); // 解析 JSON 数据
app.use(bodyParser.urlencoded({ extended: true })); // 解析 URL 编码的数据

// 定义 POST 路由
app.post('/submit', (req, res) => {
  // 从请求体中获取数据
  const { name, age } = req.body;
  res.send(`Received name: ${name}, age: ${age}`);
});

// 启动服务器
app.listen(PORT, () => {
  console.log(`Server is running at http://localhost:${PORT}`);
});

  1. 响应设置：
  1. 兼容原生 HTTP 模块的获取方式：
  1. res.statusCode = 404;
  2. res.statusMessage = 'xxx';
  3. res.setHeader('abc','xyz');
  4. res.write('响应体');
  5. res.end('xxx');
  1. express 封装的响应的方法：
  1. res.status(500);：设置响应状态码；
  2. res.set('xxx','yyy');：设置响应头；
  3. res.send('中文响应不乱码');：设置响应体；
  4. res.status(404).set('xxx','yyy').send('你好')：连贯操作
  1. 其他响应：
  1. res.redirect('路径');：重定向；
  2. res.download('./package.json');：下载响应；
  3. res.json();：响应 JSON；
  4. res.sendFile(__dirname + '/home.html') ;：响应文件内容

 

路由模块化 ​

TIP

  • 路由模块化-目录结构：
plaintext
project
│   app.js           # 主应用文件
└───routes
    ├── users.js     # 用户相关的路由
    └── products.js  # 产品相关的路由
  • 实现路由模块化
js
const express = require('express');
const app = express();
const PORT = 3000;

// 引入路由模块
const userRoutes = require('./routes/users');
const productRoutes = require('./routes/products');

// 使用路由模块
app.use('/users', userRoutes);
app.use('/products', productRoutes);

app.listen(PORT, () => {
  console.log(`Server is running at http://localhost:${PORT}`);
});

中间件 ​

TIP

  1. 定义：
  1. 中间件（Middleware）本质是一个回调函数；
  2. 中间件函数可以像路由回调一样访问 请求对象（request），响应对象（response） 🔗

  1. 作用：中间件的作用就是使用函数封装公共操作，简化代码

  2. 中间件类型：

  1. 全局中间件：应用级别的中间件，它会对所有请求生效
  1. 说明：通过 app.use() 方法可以定义全局中间件，所有请求都会经过它

  1. 使用：
js
const express = require('express');
const app = express();

// 全局中间件 - 打印请求方法和路径
app.use((req, res, next) => {
  console.log(`${req.method} ${req.path}`);
  next(); // 调用 next() 继续到下一个中间件或路由
});

// 全局中间件 - 错误处理的全局中间件
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});

app.get('/', (req, res) => {
  res.send('首页');
});

app.get('/about', (req, res) => {
  res.send('关于页面');
});

app.listen(3000, () => {
  console.log('Server is running on http://localhost:3000');
});

  console.log(`${req.method} ${req.path}`);

const app = express();

  1. 路由中间件：
  1. 说明：只需要对某一些路由进行功能封装 ，则就需要路由中间件
  2. 调用格式：app.get('/路径', 中间件1, 中间件2, (req,res)=>{ });
  3. 使用：
js
// auth.js - 简单的认证中间件
const auth = (req, res, next) => {
  const token = req.headers['authorization'];

  if (token === 'valid-token') {
    next(); // 如果认证成功，继续处理请求
  } else {
    res.status(403).send('Forbidden: Unauthorized access'); // 认证失败，返回 403 错误
  }
};

module.exports = auth;

  1. 其他中间件：
  1. 静态资源中间件：
  1. 配置：app.use(express.static(path.join(__dirname, 'public')))
  2. 作用：将当前文件夹下的public目录作为网站的根目录
  3. 注意：
  • index.html 文件为默认打开的资源
  • 如果public目录下有index.html文件，并且又设置了index.html的路由，那么谁书写在前，优先执行谁
  • 路由响应动态资源，静态资源中间件响应静态资源
  1. express-fileupload 中间件：
  1. 安装：npm i express-fileupload
  2. 导入：const fileUpload = require('express-fileupload')
  3. 应用：app.use(fileUpload())
  4. 作用：主要用于文件上传
  1. cors（允许跨域） 中间件：
  1. 安装：cnpm i cors --save
  2. 导入：const cors = require('cors')
  3. 应用：app.use(cors())
  4. 作用：设置允许跨域请求

 

  1. 注意：
  • 一定要在路由之前注册中间件（错误级别中间件要在最后）；
  • 客户端发送过来的请求，可以连续调用多个中间件进行处理；
  • 执行完中间件的业务代码之后，一定要调用next() 函数；
  • 调用完next()函数之后不要再写额外的代码；
  • 连续调用多个中间件时，多个中间件之间共享req和res对象
