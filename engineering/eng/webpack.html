webpack 打包工具 ​

webpack ​

TIP

  1. 背景：浏览器只认识 html、css、js、图片等资源

  2. 作用：开发的时候，把浏览器不认识的文件打包成浏览器认识的文件，这样浏览器就能运行这些文件了

  • 具体来说，就是以某个模块作为入口，根据入口分析出所有模块的依赖关系，然后对各种模块进行合并、压缩，形成最终的打包结果
  • 在 webpack 的世界中，一切皆是模块

webpack 的几个点 ​

TIP

1. 页面模板 ​

  1. 对于单页应用而言，只有一个空白的页面，所有内容都靠 JS 代码创建

  2. webpack 会自动生成一个页面，并且在页面中会自动加入对 js 和 css 的引用

  3. 它生成页面时，参考的是public/index.html，其称之为页面模板

 

2. public 目录 ​

  • webpack 会非常暴力的将 public 目录中的所有文件（除页面模板外），复制到打包结果中

 

3. 开发服务器 ​

  1. 在开发阶段，我们可以运行npm run serve命令，该命令会让webpack启动一个开发服务器

  2. 在这个阶段，webpack 并不会形成打包结果文件，而是把打包的内容放到内存中，当我们请求服务器时，服务器从内存中给予我们打包结果

  3. 与此同时，当源码发生变动时，webpack 会自动重新打包，同时刷新页面以访问到最新的打包结果

 

4. 文件缓存 ​

  1. 打包后的文件名会以哈希值作为后缀命名；源码内容不变，hash 不变；源码内容变化，hash 变化

  2. 之所以这样做，是因为生产环境中，浏览器会对除页面外的静态资源进行缓存；

  3. 如果不设置 hash 值，一旦代码更新，浏览器还会使用之前缓存的结果，无法使用最新的代码

 

5. 资源路径 ​

  1. 除代码和样式模块外，其他模块被视为资源模块；
  • 资源模块在源代码中的路径和打包后的路径是不一样的，这就导致我们在编写代码的时候，根本无法知晓最终的路径
  1. 比如：
  • 在 css 中使用背景图片时，webpack 在打包时，会将其自动转换为打包结果的路径
css
.container{
  /* 背景图使用了源码中的路径 */
  backgroud: url('../assets/1.png');
}

8. JS 兼容性 ​

  • 通过配置 webpack，可以对 js 语法降级处理，实现浏览器兼容性
  1. babel.config.js：通过配置该文件，可以设置对哪些 js 代码进行降级处理
  2. .browserslistrc：通过配置该文件，可以设置在降级时，要兼容哪些浏览器，兼容的范围越光，降级产生的代码就越多，自然，打包后的体积就越大

 

9. 打包压缩 ​

  • 通过配置 webpack，可以对 js 和 css 代码进行压缩

 

10. css module ​

  1. 作用：当样式文件以xxx.mdoule.xxx的方式命名时，webpack 会将该文件当成一个开启了css module的文件

  • 比如：index.module.less、movie.module.css，都是开启了css module的文件

  • 文件中的所有类名都会被 hash 化：

webpack 中配置 Pluigin ​

TIP

1. webpack-dev-server 插件 ​

  1. 作用：
  1. 在修改源代码的时候页面实现自动刷新(热重载)
  2. 通过配置可以自动打开浏览器
  1. 配置：
  1. 下载包：npm i webpack-dev-server@3.11.0 -D；
  2. 在 webpack.config.js 中添加配置：
js
devServer:{
    open:true,
    host:'127.0.0.1'，
    port:8080
}
1
2
3
4
5
  1. 修改 package.json -> scripts 中的 dev 命令
js
"scripts"：{
   "dev":"webpack serve"
}
1
2
3
  1. 使用：
  1. 运行npm run dev命令，重新进行项目的打包；
  2. 在浏览器中访问 http://localhost:8080 地址，查看自动打包效果
  1. 原理：
  1. 资源(index.html、main.js)⇄(打包)托管到内存 ⇄(通知)浏览器刷新；
  2. 不配置 webpack-dev-server 的情况下，webpack 打包生成的文件，会存放到实际的物理磁盘上；
  3. 配置了 webpack-dev-server 之后，打包生成的文件存放到了内存中，托管到内存服务器中

 

2. html-webpack-plugin 插件 ​

  1. 作用：使用 HtmlWebpackPlugin 插件 自动引入打包后的资源文件到 html 文件

  2. 配置：

  1. 下载包：npm i --save-dev html-webpack-plugin；
  2. 在 webpack.config.js 中引入插件并设置基本配置；
  3. 在 webpack.config.js 中添加插件的配置
