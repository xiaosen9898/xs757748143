背景 ​

TIP

HTTP 是一种无状态的协议，它没有办法区分多次的请求是否来自于同一个客户端（无法区分用户）

 

会话控制技术-cookie ​

TIP

  1. 概念：cookie 是 HTTP 服务器发送到用户浏览器并保存在本地的一小块数据
  2. 特点：
  1. 浏览器向服务器发送请求时，会自动将当前域名下可用的 cookie 设置在请求头中，然后发送给服务器，这个请求头的名字也叫 cookie；
  2. 没有设置生命周期的话，会在浏览器窗口关闭后消除；
  3. 自身域名下的cookie只能用于自身；
  4. cookie值大小限制在4kb；
  5. cookie不具有安全性
  1. cookie 操作：
  1. 浏览器操作；
  2. express中设置cookie：
  1. 设置：在路由规则中（假设是登陆路由规则），响应cookie
js
app.post('/login', (req, res) => {
    // 前置操作
    res.cookie('key', 'value')
    res.send('home')
})

// 设置有生命周期的cookie
app.post('/login', (req, res) => {
    // 前置操作
    req.cookie('key', 'value', {maxAge: 60 * 1000}) //
    res.send('home')
})

会话控制技术-session ​

TIP

  1. 概念：
  • session 是保存在 服务器端的一块儿数据 ，保存当前访问用户的相关信息
  • 同一个浏览器发起的多次请求，同属于一次会话（Session）

  1. 作用：实现会话控制，可以识别用户的身份，快速获取当前用户的相关信息

  2. 形式：生成的session对象以响应cookie的形式发给客户端

  3. 操作：

  1. 安装中间件：
  1. session中间件：npm i express-session connect-mongo
  • express-session：数据存到内存
  • connect-mongo：数据存到数据库
  1. 当配置完成之后，即可通过req.session来访问和使用session对象
  1. 使用：
js
/* 1. 导入中间件 */
const session = require('express-session')
const MongoStore = require('connect-mongo')

/* 2. 注册中间件 */
app.use(
    name: sid,
    secret: '秘钥',
    saveUninitialized: false, // 每次请求都设置一个cookie来存储
    resave: true, // 每次请求时重新保存session
    store: MongoStore.create({
        mongoUrl: 'mongo://127.0.0.1:27017/数据库名称'
    }),
    cookie: {
        httpOnly: true,  // 开启后无法通过 js 操作
        maxAge: 1000 * 300 // sessionID过期时间
    },
)

会话控制技术-token ​

TIP

  1. 概念：token是服务端生成并返回给HTTP客户端的一串加密字符串，token中保存着用户信息
  2. 作用：实现会话控制，可以识别用户的身份
  3. 特点：
  1. 服务端压力更小（数据存储在客户端）；
  2. 相对更安全（数据加密，可以避免 CSRF（跨站请求伪造））；
  3. 扩展性更强（服务间可以共享，增加服务节点更简单）
  1. 流程：
  1. 客户端填写账号密码，发送至服务器端；
  2. 账号密码正确，服务器端响应一个token给客户端（token一般在响应体）；
  3. 客户端根据返回的token添加在请求头中，去请求服务器需要token的资源
  1. JWT：
  1. 概念：
  • 是目前最流行的跨域认证解决方案，可用于基于 token 的身份验证；
  • JWT 使 token 的生成与校验更规范
  1. 组成：
  • 以 . 分为三个部分（Header、Payload、Signature）；
  • Payload是真正的用户信息；
  • Header和Signature是安全性相关的部分
  1. 操作：
  1. 安装：npm i jsonwebtoken；
  2. 导入：const jwt = require('jsonwebtoken')；
  3. 生成token：
js
let token = jwt.sign(
   { username: 'zhangsan', },
    '加密字符串',
    { expiresIn: 60,   } // 秒
)

session 和 cookie 的区别 ​

TIP

  1. 存在的位置：
  • cookie：浏览器端
  • session：服务端
  1. 安全性：
  • cookie 是以明文的方式存放在客户端的，安全性相对较低
  • session 存放于服务器中，所以安全性相对较好
  1. 网络传输量：
  • cookie 设置内容过多会增大报文体积，会影响传输效率
  • session 数据存储在服务器，只是通过 cookie 传递 id，所以不影响传输效率
  1. 存储限制：
  • 浏览器限制单个 cookie 保存的数据不能超过 4K ，且单个域名下的存储数量也有限制
  • session 数据存储在服务器中，所以没有这些限制
