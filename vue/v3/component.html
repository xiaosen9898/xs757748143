2. 子传父 Emit ​

  1. 子组件使用defineEmit()获取 emit 对象，通过 emit 触发事件，并且传递数据；
  2. 父组件监听子组件的事件，接收参数
vue
<template>
    <h2>子组件</h2>
    <!-- 按钮点击时触发 custom-event 事件 -->
    <button @click="changeCount">向父组件传递数据</button> 
</template>

<script setup>
// 通过 emit 触发事件，将数据传递给父组件
// defineEmits(['change-count']) 用于声明该组件会触发一个名为 change-count 的事件
const emit = defineEmits(['change-count'])                
// 子组件的方法：触发事件并传递数据
const changeCount = () => {                               
  emit('change-count', 1000);                             
};                                                        

// *指定事件的参数类型
const emit = defineEmits<{
 (event: 'update', payload: string): void;
 (event: 'delete', id: number): void;
}>();
const sendUpdate = () => {
 emit('update', 'New Value');
};
const sendDelete = () => {
 emit('delete', 123);
};

</script>

3. v-model ​

  1. 场景：父 ⇆ 子
  2. 本质：
  1. 组件标签上的v-model的本质：:moldeValue ＋ update:modelValue事件
  2. 如果 v-model 后面不使用自定变量 v-model，那么本质上父传的就是 :modelValue，监听的事件为@update:modelValue=""
  3. 如果 v-model 后面自定了变量 v-model:ss，那么本质上父传的就是 :ss，监听的事件为@update:ss=""
vue
<!-- ====== 父组件 ====== -->
<template>
  <!-- 组件标签上使用v-model指令 -->
  <Child v-model="username" />

  <!-- 组件标签上v-model的本质 -->
  <!-- <Child  :modelValue="userName" @update:model-value="userName = $event"/> -->
</template>

<script setup>
import { ref } from 'vue'
import Child from './components/Child.vue'
const username = ref('admin')
</script>


<!-- ====== 子组件 ====== -->
<template>
  <div class="box">
    <!-- 将接收的value值赋给input元素的value属性，目的是：为了呈现数据 -->
    <!-- 给input元素绑定原生input事件，触发input事件时，进而触发update:model-value事件-->
    <input :value="modelValue" @input="emit('update:model-value', $event.target.value)" />
    <!-- <input v-model="modelValue" />  会报错 -->
  </div>
</template>

<script setup lang="ts">
// 接收props
defineProps(['modelValue'])
// 声明事件
const emit = defineEmits(['update:model-value'])
</script>

}

js
// 引入mitt
import mitt from "mitt";

// 创建emitter
const emitter = mitt()

/*
  // 绑定事件
  emitter.on('abc',(value)=>{

7. provide、inject ​

  1. 场景：爷 → 孙
  2. 使用：
  1. 在祖先组件中通过provide配置向后代组件提供数据
  2. 在后代组件中通过inject配置来声明接收数据
  1. 注意：传 ref 声明的数据时，应传响应式数据，也就是说不应该传.value
vue
<!-- ====== 爷组件 ====== -->
<template>
  <div class="father">
    <h3>父组件</h3>
    <h4>资产：{{ money }}</h4>
    <h4>汽车：{{ car }}</h4>
    <button @click="money += 1">资产+1</button>
    <button @click="car.price += 1">汽车价格+1</button>
    <Child/>
  </div>
</template>

<script setup lang="ts" name="Father">
  import Child from './Child.vue'
  import { ref,reactive,provide } from "vue";
  // 数据
  let money = ref(100)
  let car = reactive({
    brand:'奔驰',
    price:100
  })
  // 用于更新money的方法
  function updateMoney(value:number){
    money.value += value
  }
  // 提供数据
  provide('moneyContext',{money,updateMoney})
  provide('car',car)
</script>

Keep-Alive ​

  1. 作用：<KeepAlive> 是 Vue 提供的一个内置组件，用来缓存组件实例，防止组件重复销毁和创建

  2. 工作机制：

  1. 第一次渲染：正常挂载组件
  2. 被切换出去（例如被 v-if 隐藏或路由切换）：不会销毁实例，而是缓存
  3. 再次显示：直接复用缓存的实例，跳过重新创建流程
  1. 属性：
  1. include：指定需要缓存的组件名称
  2. exclude：指定不需要缓存的组件名称
  3. max：指定最大缓存组件数；如果缓存的实例数量超过指定的那个最大数量，则最久没有被访问的缓存实例将被销毁
  1. 使用：
vue
<template>
  <KeepAlive>
    <component :is="currentComponent" />
  </KeepAlive>
</template>

<script setup>
import Foo from './Foo.vue'
import Bar from './Bar.vue'
import { ref } from 'vue'

const currentComponent = ref('Foo') // 可切换为 'Bar'
</script>
