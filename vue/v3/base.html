setup 函数 ​

TIP

  1. 说明：
  1. 比 beforeCreate 还要早执行的钩子函数；
  2. 是 componentAPI 的入口；
  3. 不能使用 this，this 指向 undefined；
  4. template 模板中使用的数据，要在 setup 函数返回

 

  1. setup 语法糖：
  1. 语法：<script setup> ... </script>
  2. 好处：
  1. 顶层的绑定会自动暴露给模板，所以定义的变量、函数和 import 导入的内容都可以直接在模板中直接使用；
  2. 支持顶层 await，在<script setup></script> 顶层中，无需写async，直接写await即可；
  3. 无需手动注册局部组件，引入即可使用；
  4. 使用了语法糖之后，不需要再写：
js
 export default {
   setup() {  return {} }
 }

3. customRef ​

作用：创建一个自定义的ref，并对其依赖项跟踪和更新触发进行逻辑控制

使用：

ts
// 编写一个hooks（实现防抖效果）
import {customRef } from "vue";

export default function(initValue: string, delay: number){
  let msg = customRef((track,trigger) => {
    let timer:number
    return {
      get(){
        track() // 告诉Vue数据msg很重要，要对msg持续关注，一旦变化就更新
        return initValue
      },
      set(value){
        clearTimeout(timer)
        timer = setTimeout(() => {
          initValue = value
          trigger() //通知Vue数据msg变化了
        }, delay);
      }
    }
  })
  return { msg }
}

raw 原始数据 ​

TIP

1. toRaw ​

  1. 作用：用于获取一个响应式对象的原始对象， toRaw 返回的对象不再是响应式的，不会触发视图更新
  2. 使用：
vue
<script setup lang="ts">
  import { reactive,toRaw,markRaw,isReactive } from "vue";

  /* toRaw */
  // 响应式对象
  let person = reactive({name:'tony',age:18})
  // 原始对象
  let rawPerson = toRaw(person)


  /* markRaw */
  let citysd = markRaw([
    {id:'asdda01',name:'北京'},
    {id:'asdda02',name:'上海'},
    {id:'asdda03',name:'天津'},
    {id:'asdda04',name:'重庆'}
  ])
  // 根据原始对象citys去创建响应式对象citys2 —— 创建失败，因为citys被markRaw标记了
  let citys2 = reactive(citys)
  console.log(isReactive(person))
  console.log(isReactive(rawPerson))
  console.log(isReactive(citys))
  console.log(isReactive(citys2))
</script>

toRefs ​

  1. 背景：如果对一个响应数据，进行解构或者展开，会丢失他的响应式特性

  2. 作用：将一个响应式对象转为一个普通对象，普通对象的每一个属性对应的是一个 ref 值

// 使用toRefs之后需要 .value

</script>

vue 指令 ​

TIP

定义：特殊的 html 标签属性（特点: v- 开头）

作用：每个 v- 开头的指令, 都有着自己独立的功能, 将来 vue 解析时, 会根据不同的指令提供不同的功能

1. v-bind ​

  1. 语法：v-bind:属性名 = "值"
  2. 简写：:属性名 = "值"
  3. 作用：绑定标签的属性，使其值变成变量形式

 

2. v-on ​

  1. 语法 1：v-on:事件类型 = "执行语句"，简写：@事件类型 = "执行语句"
  2. 语法 2：v-on:事件类型 = "函数名(参数)"，简写：@事件类型 = "函数名(实参)"
  3. 作用：给标签绑定事件
  4. 其他：监听多个事件

 

3. v-show ​

  1. 语法 1：v-show="布尔值" （true 显示, false 隐藏）
  2. 作用：控制盒子的显示/隐藏
  3. 原理：实质是在控制元素的 css 样式，display: none;
  4. 应用场景：如果是频繁的切换显示隐藏, 用v-show

 

4. v-if ​

  1. v-if：
  1. 语法：v-if="布尔值" （true 显示, false 隐藏）
  2. 作用：控制盒子的显示隐藏
  3. 原理：实质是在动态的创建/删除元素节点
  4. 应用场景：如果是不用频繁切换, 要么显示, 要么隐藏的情况, 适合于用 v-if
  1. v-else：
  1. 语法：v-else="布尔值" （true 显示, false 隐藏）
  2. 作用：控制盒子的显示隐藏
  1. v-else-if：
  1. 语法：v-else-if="布尔值" （true 显示, false 隐藏）
  2. 作用：控制盒子的显示隐藏

 

5. v-for ​

  1. 循环类型：
js
v-for = "(item，index) in 数组" :key = "item.id"

  1. 说明：

  1. key 属性主要是用于优化虚拟 DOM 的渲染性能的，相当于是给虚拟 DOM 元素一个唯一性标识

  2. 使用 v-for 循环渲染的时候也可以使用 template 来循环多个元素，此时 key 就挂在 template 标签上面

  3. v-for 也是存在作用域的，作用域的工作方式和 JS 中的作用域工作方式类似

  4. vue3 中 v-if 优先级高于 v-for

6. v-model ​

  1. 语法：v-model = "值"
  2. 作用：双向数据绑定，给表单元素使用，数据 ⇄ 视图
  3. 原理：
  1. 页面渲染时，input 的 value 会绑定到 msg
  2. 用户输入内容时，触发 input 事件，将 event.target.value 赋值给 msg
  3. 由于 msg 是 ref（或 reactive）响应式数据，Vue 会自动重新渲染视图
vue
<input v-model="msg" />

<input
  :value="msg"
  @input="msg = $event.target.value"
/>

1. input 或者 textarea，元素会绑定 input 事件，绑定的值是 value
2. 如果是单选框或者复选框，背后绑定的事件是 change 事件，绑定的值 checked
3. select 下拉列表绑定的也是 change 事件，绑定的值是 value

computed 计算属性 ​

TIP

  1. 作用：一个特殊属性，值依赖于另外一些数据动态计算出来，其计算出来的那个值也是响应式的

  2. 特点：

  1. 返回的值是 ref，具有响应式
  2. 计算后会立刻缓存，下次直接读缓存
  3. 依赖项改变，函数会重新执行并重新缓存
  1. 使用：
vue
<script setup>
   const 变量名 = computed( ()  =>  {} )
</script>
1
2
3
vue
<script setup>
   const 变量名 = computed({
      get(){ return xxx },
      set(value) { }
   })
</script>

// 1. get为计算逻辑，set为修改逻辑
// 2. 当修改计算属性，会触发set函数
// 3. get方法必须返回一个值，这个值会返回给“属性名”
// 4. get方法的值会作为set方法的实参
1

侦听器 ​

TIP

watch ​

  1. 作用：在依赖的数据发生变化的时候，允许做一些副作用的操作，例如更改 DOM、发送异步请求
  1. 能监听 ref、reactive 定义的数据
  2. 能监听计算属性
  3. 能监听一个 Getter 函数（监听的是函数的返回值），监听器会执行这个函数
  4. 能监听一个包含上述内容的数组

 

  1. 特点：

没有缓存的说法 侦听器和计算属性一样，可以创建多个侦听器

 

  1. 使用：
  1. 基本语法：
vue
<script setup>
 watch(侦听对象, (newVal, oldVal) => { }, { deep: true,   immediate: true})
</script>
1
2
3
vue
<script setup>
 watch( [监听源1, 监听源2, ...], (newVal, oldVal) => {}, { deep: true } )
</script>
1
2
3
  1. 参数：接收三个参数：1 是侦听数据源，2 是侦听回调，3 是额外的配置对象
  2. 额外配置对象：
  1. immediate: true：让侦听器立即执行
  2. once: true：侦听器的回调函数只执行一次
  3. deep: true：深度监听(reactive 默认开启，且不能关闭)，监听深度嵌套的对象或数组中的 property 仍需写上
  1. 停止监视：watch 函数返回的是一个函数，调用此函数即停止监听

同步设置的侦听器会自动停止，但异步设置的侦听器不会自动停止，需要手动停止

vue
<script setup>
  const unwatch = watchEffect(() => {})
  // 手动停止
  unwatch();
</script>

watchEffect ​

  1. 作用：自动监听回调里面的响应式依赖

  2. 区别：

  1. watch 要明确指出监听的数据源，watachEffect 自动追踪响应式依赖
  2. watch 第一次默认不执行，watchEffect 会立即调用一次函数
  3. 侦听一个嵌套的数据结构的属性，watchEffect 只会侦听回调中用到的属性，而不是递归侦听所有的属性
  1. 示例：
vue
<template>
 <div class="person">
   <h1>需求：水温达到50℃，或水位达到20cm，则联系服务器</h1>
   <h2 id="demo">水温：{{temp}}</h2>
   <h2>水位：{{height}}</h2>
   <button @click="changePrice">水温+1</button>
   <button @click="changeSum">水位+10</button>
 </div>
</template>

<script lang="ts" setup name="Person">
 import {ref,watch,watchEffect} from 'vue'
 // 数据
 let temp = ref(0)
 let height = ref(0)

 // 方法
 function changePrice(){
   temp.value += 10
 }
 function changeSum(){
   height.value += 1
 }

 // 用watch实现，需要明确的指出要监视：temp、height
 watch([temp,height],(value)=>{
   // 从value中获取最新的temp值、height值
   const [newTemp,newHeight] = value
   // 室温达到50℃，或水位达到20cm，立刻联系服务器
   if(newTemp >= 50 || newHeight >= 20){
     console.log('联系服务器')
   }
 })

 // 用watchEffect实现，不用
 const stopWtach = watchEffect(()=>{
   // 室温达到50℃，或水位达到20cm，立刻联系服务器
   if(temp.value >= 50 || height.value >= 20){
     console.log(document.getElementById('demo')?.innerText)
     console.log('联系服务器')
   }
   // 水温达到100，或水位达到50，取消监视
   if(temp.value === 100 || height.value === 50){
     console.log('清理了')
     stopWtach()
   }
 })
</script>

nextTick ​

TIP

  1. 背景：Vue 在数据变化后，DOM 更新是异步的
  2. 作用：等 DOM 更新后, 才会触发执行此方法里的函数体
  3. 使用：
vue
<script setup>
import { nextTick } from 'vue'
setTimeout(async () => {
  await nextTick()
  // ...
}, 3000)
</script>
1
2
3
4
5
6
7
  1. 原理：异步任务调度机制

实现关键：Vue 内部通过统一的调度器 scheduler.ts 管理任务队列，并且通过以下机制实现异步调度

  1. Promise.then()
  2. MutationObserver
  3. setImmediate()
  4. setTimeout()

具体实现：Vue 3 的 nextTick() 就是想找一个 “尽快执行，但又等 DOM 更新完” 的方法

js
if (Promise 可用) {
  用 Promise.then() 来实现 nextTick
} else if (MutationObserver 可用) {
  用 MutationObserver
} else if (setImmediate 可用) {
  用 setImmediate
} else {
  用 setTimeout 兜底
}

总结：Vue 会优先选择 最快的异步方式 来实现 nextTick，比如 Promise.then()，这样能保证你的回调尽早执行，但又是在 DOM 更新之后

 

数组的侦测 ​

TIP

  1. 两种数组方法：
  1. 变更方法：调用这些方法时会对原来的数组进行变更

push、pop、shift、unshift、splice、sort、reverse

  1. 非变更方法：调用这些方法不会对原来的数组进行变更，而是会返回一个新的数组

filter、concat、slice、map

  1. 响应式处理：
  1. 针对变更方法，数组只要一更新，就会触发它的响应式，页面会重新渲染
  2. 如果是非变更方法，那么需要使用方法的返回值去替换原来的值
