setup 函数 ​

TIP

  1. 说明：
  1. 比 beforeCreate 还要早执行的钩子函数；
  2. 是 componentAPI 的入口；
  3. 不能使用 this，this 指向 undefined；
  4. template 模板中使用的数据，要在 setup 函数返回

 

  1. setup 语法糖：
  1. 语法：<script setup> ... </script>
  2. 好处：
  1. 顶层的绑定会自动暴露给模板，所以定义的变量、函数和 import 导入的内容都可以直接在模板中直接使用；
  2. <script setup></script> 中，无需写async，直接写await即可；
  3. 使用了语法糖之后，不需要再写：
js
 export default {
     setup() {  return {} }
 }

响应式数据 ​

TIP

1. reactive ​

  1. 作用：声明复杂数据类型，使其为响应式数据

  2. 使用：

vue
<script setup>
  import { reactive } from 'vue'
  const user = reactive({ name: 'zs', age:20 })  
</script>

toRefs ​

TIP

  1. 背景：如果对一个响应数据, 进行解构或者展开，会丢失他的响应式特性

  2. toRefs 作用：对一个响应式对象的所有内部属性，都做响应式处理

  3. 使用：

vue
<script setup>
import { reactive, toRefs } from 'vue'
const user = reactive({
  name: '张三',
  age: 18,
})
const { name, age } = toRefs(user)

// 使用toRefs之后需要 .value
console.log(name.value)
</script>

computed 计算函数 ​

TIP

  1. 作用：一个特殊属性, 值依赖于另外一些数据动态计算出来，其计算出来的那个值也是响应式的

  2. 特点：

  1. 计算后会立刻缓存，下次直接读缓存
  2. 依赖项改变, 函数会重新执行并重新缓存
  1. 使用：
vue
<script setup>
   const 变量名 = computed( ()  =>  {} )
</script>

watch 侦听器 ​

TIP

  1. 作用：
  1. 只能监听 ref、reactive 定义的数据
  2. 只能监听函数返回一个值
  3. 只能监听一个包含上述内容的数组

 

  1. 特点：

侦听器和计算属性一样，可以创建多个侦听器

 

  1. 使用：
  1. 基本语法：
vue
<script setup>
   watch(侦听对象, (newVal, oldVal) => { }, { deep: true,   immediate: true})
</script>
1
2
3
  1. 参数：接收三个参数：1 是侦听数据源，2 是侦听回调，3 是额外的配置
  2. 注意：
  1. deep: true 深度监听(reactive 默认开启，且不能关闭)，监听深度嵌套的对象或数组中的 property 仍需写上
  2. immediate: true 让侦听器立即执行
  1. 停止监视：watch 函数返回的是一个函数，调用此函数即停止监听

 

  1. 看情况监视：
vue
<script setup>
  // 1. 监听基本类型数据
  let num = ref(1)
  watch( num, (newVal, oldVal) => { } ) // 监视源不能写 .value

  // 2. 监听一个对象（监听的是引用地址）
  const person = ref({ name: 'jack', age: 10 })
  watch( person, (newVal, oldVal) => {} )

  // 注意：
  //  1. 此时监听的是对象的地址，若想监听对象内部属性的变化，则要开启深度监听watch( person, (newVal, oldVal) => {}, { deep: true })
  //  2. 如果监听的是整个对象的话，修改里边某个属性（引用没变），旧值和新值一样，修改整个对象（引用改变），新旧值才不一样

</script>

watchEffect ​

TIP

  1. 作用：
  1. 监听响应式数据的变化（监听数据变化的方式不同）
  2. 会立即调用一次函数
  1. 区别：
  1. watch 要明确指出监听的数据源
  2. watchEffect 不需要明确指出监听的数据源（函数中用到哪些数据就监听哪些数据）
vue
<template>
 <div class="person">
   <h1>需求：水温达到50℃，或水位达到20cm，则联系服务器</h1>
   <h2 id="demo">水温：{{temp}}</h2>
   <h2>水位：{{height}}</h2>
   <button @click="changePrice">水温+1</button>
   <button @click="changeSum">水位+10</button>
 </div>
</template>

<script lang="ts" setup name="Person">
 import {ref,watch,watchEffect} from 'vue'
 // 数据
 let temp = ref(0)
 let height = ref(0)

 // 方法
 function changePrice(){
   temp.value += 10
 }
 function changeSum(){
   height.value += 1
 }

 // 用watch实现，需要明确的指出要监视：temp、height
 watch([temp,height],(value)=>{
   // 从value中获取最新的temp值、height值
   const [newTemp,newHeight] = value
   // 室温达到50℃，或水位达到20cm，立刻联系服务器
   if(newTemp >= 50 || newHeight >= 20){
     console.log('联系服务器')
   }
 })

 // 用watchEffect实现，不用
 const stopWtach = watchEffect(()=>{
   // 室温达到50℃，或水位达到20cm，立刻联系服务器
   if(temp.value >= 50 || height.value >= 20){
     console.log(document.getElementById('demo')?.innerText)
     console.log('联系服务器')
   }
   // 水温达到100，或水位达到50，取消监视
   if(temp.value === 100 || height.value === 50){
     console.log('清理了')
     stopWtach()
   }
 })
</script>

ref ​

TIP

  1. 作用：获取 DOM 元素和组件实例

  2. 使用：

vue
<script setup>
 import { ref } from 'vue'
 // 定义 ref
 const myRef = ref(null)

 // 调用子组件的sayHi方法
 const sayHello = () => {
   myRef.value.sayHi()
 }

 // 挂载时调用子组件的sayHi方法
 onMounted(() => {
   myRef.value.sayHi()
 })
</script>
<template>
  <!-- 在子组件标签上绑定ref -->
  <Son ref="myRef"></Son>
</template>
