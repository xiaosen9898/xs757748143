缓存事件处理函数 ​

TIP

  1. VUE 认为事件处理函数是不会变的

// vue2
render(ctx){
  return createVNode("button", {
    onClick: function($event){
      ctx.count++;
    }
  })
}

// vue3
render(ctx, _cache){
  return createVNode("button", {
    onClick: cache[0] || (cache[0] = ($event) => (ctx.count++))
  })
}
</script>

Block Tree ​

TIP

vue2 在对比新旧树的时候，并不知道哪些节点是静态的，哪些是动态的，因此只能一层一层比较，这就浪费了大部分时间在比对静态节点上

 

vue3 在对比新旧树时，会记录静态节点，并生成一个 block 树，这样当对比新旧树时，就可以直接跳过静态节点，从而提高效率

 

PatchFlag ​

TIP

vue2 在对比每一个节点时，并不知道这个节点哪些相关信息会发生变化，因此只能将所有信息依次比对

vue3 在对比每一个节点时，编译器会记录节点的动态内容，并生成一个 PatchFlag

为什么去掉？

vue2 的全局构造函数带来了诸多问题：

  1. 调用构造函数的静态方法会对所有 vue 应用生效，不利于隔离不同应用
  2. vue2 的构造函数集成了太多功能，不利于 tree shaking，vue3 把这些功能使用普通函数导出，能够充分利用 tree shaking 优化打包体积
  3. vue2 没有把组件实例和 vue 应用两个概念区分开，在 vue2 中，通过 new Vue 创建的对象，既是一个 vue 应用，同时又是一个特殊的 vue 组件。vue3 中，把两个概念区别开来，通过 createApp 创建的对象，是一个 vue 应用，它内部提供的方法是针对整个应用的，而不再是一个特殊的组件。

数据响应式 ​

TIP

vue2 和 vue3 均在 beforeCreate 之后，在 created 之前创建数据响应式，但做法不一样

vue2 使用Object.defineProperty，递归创建数据响应式

vue3 使用Proxy，直接创建数据响应式

 

谈谈你对 vue3 数据响应式的理解？

  1. vue3 不再使用 Object.defineProperty 的方式定义完成数据响应式，而是使用 Proxy；
  2. 除了 Proxy 本身效率比 Object.defineProperty 更高之外，由于不必递归遍历所有属性，而是直接得到一个 Proxy。所以在 vue3 中，对数据的访问是动态的，当访问某个属性的时候，再动态的获取和设置，这就极大的提升了在组件初始阶段的效率；
  3. Proxy 可以监控到成员的新增和删除，因此，在 vue3 中新增成员、删除成员、索引访问等均可以触发重新渲染，而这些在 vue2 中是难以做到的。

v-model ​

TIP

  1. vue2 提供了两种双向绑定：v-model和.sync，在 vue3 中，去掉了.sync 修饰符，只需要使用一个或多个 v-model 进行双向绑定即可。
vue
<!-- vue2 -->
<ChildComponent v-model="pageTitle" />
<!-- 等价于 -->
<ChildComponent :value="pageTitle" @input="pageTitle = $event" />

<!-- vue3 -->
<ChildComponent v-model="pageTitle" />
<!-- 等价于 -->
<ChildComponent :modelValue="pageTitle" @update:modelValue="pageTitle = $event" />

<!-- 使用多个v-model -->
<MyInput v-model:a="userName" v-model:b="userPsw"/>
<!-- 等价于 -->
<MyInput :a="userName" @update:a="userName = $event" :b="userPsw" @update:b="userPsw = $event"  />

  1. 在自定义组件中使用 v-model 修饰符，允许自定义 v-model 修饰符，需自己实现
vue
<!-- ====== 父组件 ====== -->
<template>
  <div class="app-container">
    <Child v-model:text.trim="cont" />
  </div>
</template>

<script setup>
import { ref } from 'vue'
import Child from './components/Child.vue'

const cont = ref('')
</script>

<!-- ====== 子组件 ====== -->
<template>
  <input type="text" :value="text" @input="handleTextChange" />
</template>

<script setup>
import { defineProps, defineEmits } from 'vue'

// 定义 props
const props = defineProps({
  text: {
    type: String,
    default: ''
  },

  textModifiers: {
    default: () => ({})
  }
})

// 定义 emits
const emit = defineEmits(['update:text'])

// 文本改变处理函数
const handleTextChange = (e) => {
  console.log('textModifiers--->', props.textModifiers.trim) // true
  if (props.textModifiers && props.textModifiers.trim) {
    e.target.value = e.target.value.trim()
  }
  emit('update:text', e.target.value)
}
</script>

响应式数据的变化 ​

响应式数据 ​

TIP

  1. ref：既可以以是基本数据类型，也可以是对象类型
  1. 对.value 的访问是响应式的
  2. 如果给的值是一个对象，则会通过 reactive 函数进行代理
  3. 如果已经是代理，则直接使用代理

  1. reactive：用于声明对象类型，深度代理对象中的所有成员

  2. readonly：用于声明只能读取代理对象，值不可修改

 

计算属性 ​

TIP

computed：是响应式的，当读取 value 值时，会根据情况决定是否要运行函数

 

监听 ​

TIP

  1. watch：用于监听对象，当对象中的值发生变化时，会执行回调函数

  2. watchEffect：依赖项改变，会执行回调函数

 

toRef ​

TIP

  1. toRef：得到一个响应式对象某个属性的 ref 格式

  2. toRefs：把一个响应式对象的所有属性转换为 ref 格式，然后包装到一个 plain-object 中返回
