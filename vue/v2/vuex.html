VUEX ​

概念 ​

TIP

  1. 定义：状态管理工具（状态指的是数据）

  2. 思想：数据集中管理集中维护

 

配置 ​

TIP

  1. 安装：npm i vuex@3
  2. 配置：
js
// src/store/index.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    },
    decrement(state) {
      state.count--;
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment');
    },
    decrement({ commit }) {
      commit('decrement');
    }
  },
  getters: {
    doubleCount(state) {
      return state.count * 2;
    }
  }
});

核心-state ​

TIP

  1. 定义：理解为数据
  2. 特点：
  1. 数据刷新会丢失；
  2. 一般结合 localStorage 来使用

  1. 注意：修改 state 的唯一方式是提交一个 mutation，不能直接修改 state

  2. 使用：

  1. store 不分模块：
plaintext
project
│
├── src
│   ├── store
│   │   └── index.js  // 这里定义 Vuex store
│   └── components
│       └── MyComponent.vue

核心-getters ​

TIP

  1. 定义：状态 state 的计算属性
  2. 特点：
  • 具有缓存功能；
  1. 使用：
  1. store 不分模块：
js
// src/store/index.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
  state: {
    count: 0,
    user: {
      name: 'Alice',
      age: 25
    }
  },
  getters: {                                  
    userName: (state) => state.user.name,     
    userAge: (state) => state.user.age,       
    doubleCount: (state) => state.count * 2
  },                                          
  mutations: {
    increment(state) {
      state.count++;
    }
  }
});

核心-mutations ​

TIP

  1. 定义：Vuex 修改状态的唯一方式，通过提交一个 mutation
  2. 特点：
  • 写同步代码的地方
  1. 使用：
  1. store 不分模块：
js
// src/store/index.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
  state: {
    count: 0,
    name: 'Alice'
  },
  mutations: {                   
    increment(state) {           
      state.count++;             
    },                           
    setName(state, newName) {    
      state.name = newName;      
    }                            
  }

核心-actions ​

TIP

  1. 定义：action 处理的是异步修改状态
  2. 特点：
  • mutation 修改数据是同步的，action 处理的是异步修改状态
  • 通过 dispatch 触发 action
  1. 使用：
  1. store 不分模块：
js
// src/store/index.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
  state: {
    name: 'Alice',
    user: null
  },
  mutations: {
    setName(state, newName) {
      state.name = newName;
    },
    setUser(state, user) {
      state.user = user;
    }
  },
  actions: {
    updateName({ commit }, newName) {
      setTimeout(() => {
        commit('setName', newName);
      }, 1000);
    },
    async fetchUser({ commit }) {
      const response = await fetch('https://api.example.com/user');
      const user = await response.json();
      commit('setUser', user);
    }
  }
});

模块-modules ​

TIP

  1. 定义：
  1. 仓库数据多了，就用模块来划分；
  2. 每个模块可以有自己的 state、getters、mutations、actions；
  3. 刷新的时候会重新创建一个仓库
  1. 注意：
  1. state 默认是在模块内部的，其他属性是默认放在全局仓库的；
  2. 使用命名空间namespaced: true，就能把其他属性限制在模块中
