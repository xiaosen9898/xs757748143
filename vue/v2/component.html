 name: 'ParentComponent',

子传父 ​

  1. 子组件通过this.$emit('事件名', '子组件的数据')给父组件发事件并传递参数；
  2. 父组件监听子组件的事件，接收参数；
vue
<template>
  <div>
    <button @click="sendDataToParent">点击传递数据给父组件</button>  
  </div>
</template>

<script>
export default {
  name: 'ChildComponent',
  methods: {
    sendDataToParent() {                                
      // 通过 $emit 触发事件并传递数据
      this.$emit('send-data', '这是来自子组件的数据');    
    }                                                   
  }
};
</script>

组件使用 v-model ​

TIP

原理：

  1. value：父组件通过value向子组件传递初始数据；
  2. input 事件：子组件通过$emit('input', newValue)向父组件发送更新后的数据
vue
<template>
  <div>
    <h1>父组件</h1>
    <p>输入框内容：{{ parentInput }}</p>
    <!-- 使用 v-model 绑定数据到子组件 -->
    <ChildComponent v-model="parentInput" />   
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';

export default {
  name: 'ParentComponent',
  components: {
    ChildComponent
  },
  data() {
    return {
      parentInput: '默认文本'
    };
  }
};
</script>

    }
  },
  methods: {

    }                                           
  }
};

keep-alive 组件 ​

TIP

  1. 作用：<KeepAlive> 是 Vue 提供的一个内置组件，用来缓存组件实例，防止组件重复销毁和创建

  2. 工作机制：

  1. 第一次渲染：正常挂载组件
  2. 被切换出去（例如被 v-if 隐藏或路由切换）：不会销毁实例，而是缓存
  3. 再次显示：直接复用缓存的实例，跳过重新创建流程
  1. 属性：
  1. include：指定需要缓存的组件名称
  2. exclude：指定不需要缓存的组件名称
  3. max：指定最大缓存组件数；如果缓存的实例数量超过指定的那个最大数量，则最久没有被访问的缓存实例将被销毁
  1. 使用：
vue
<template>
  <KeepAlive>
    <component :is="currentComponent" />
  </KeepAlive>
</template>

<script setup>
import Foo from './Foo.vue'
import Bar from './Bar.vue'
import { ref } from 'vue'

const currentComponent = ref('Foo') // 可切换为 'Bar'
</script>

插槽 ​

TIP

  1. 作用：在子组件中自定义某一块不确定的内容
  2. 实现：子组件中用<slot>占位，父组件中用标签替换

1. 默认插槽 ​

  1. 封装：用<slot></slot>占位
  2. 默认内容：在<slot>标签内放置内容, 作为默认显示内容，父组件不传内容时，显示默认内容，且 slot 标签不会显示出来
  3. 使用：使用插槽组件时，用别的标签替换<slot><slot/>
vue
<!-- ChildComponent.vue -->
<template>

2. 具名插槽 ​

  1. 需求：一个组件内有多处，需要外部传入标签，进行定制
  2. 封装：
  1. 使用<slot name="名字"></slot>占位（name 可以是动态的）；

    <!-- 定义具名插槽，名称为 footer -->
    <footer>
      <slot name="footer">默认页脚</slot>   
    </footer>
  </div>
</template>

<script>
export default {
  name: 'ChildComponent'
};

TIP

  1. 背景：每个组件都会创建自己的实例，每个实例互不影响
  2. $获取：

  2. this.$el：获取当前组件的节点
  1. 获取 dom：ref
  1. 用在标签上，获取绑定的标签
  2. 用在组件上，获取组件实例
  1. 绑定：ref
  2. 获取：$refs
vue
<!-- ChildComponent.vue -->
<template>
  <div>子组件内容</div>
</template>

<script>
export default {
  name: 'ChildComponent',
  methods: {
    sayHello() {

组件的样式冲突 ​

TIP

  1. 冲突：后面的组件样式会覆盖前面的样式
  2. 解决：在 style 标签上使用scoped属性，让样式只对当前组件生效
  3. 原理：
  1. 组件标签会增加一个 data-v-hash 属性；
  2. 样式会增加属性选择器 data-v-hash

注意：

  • 父组件可以控制子组件的根标签的样式
