插值表达式 ​

TIP

  1. 语法： {{ 表达式 }}
  2. 作用：使用data中的数据渲染视图
  3. 注意：
  • 使用的数据必须在 data 选项中存在
  • 能使用表达式，但是不能使用语句 if for ...
  • 不能在标签属性中使用 {{ }} 插值

 

组件选项 ​

TIP

1. name ​

  • 作用：给当前组件取一个名称

 

2. components ​

  1. 作用：某个组件中，注册子组件的地方
  2. 使用：
  1. 引入：import Comp from '路径'
  2. 注册：components:
  3. 使用：<Comp />

 

3. data ​

  • 作用：return返回的对象里，是声明变量的地方

 

4. methods ​

  1. 作用：声明函数的地方
  2. 使用：
  • 方法名() { 函数体 }
  • 方法用到data的成员时，需要this.成员（固定写法）

 

5. 计算属性computed ​

  1. 作用：一个特殊属性, 值依赖于另外一些数据动态计算出来
  2. 特点：
  • 计算后会立刻缓存，下次直接读缓存
  • 依赖项改变, 函数会重新执行 并 重新缓存
  1. 语法：
js
/* 语法1 */
computed: {
     属性名(){  
        return  "值"
}}

/* 语法2 */
computed:{
    属性名: {
       get(){  return "值"  }，
       set(参数){ 
         //修改的代码 
       }
    }
}
// 1. get为计算逻辑，set为修改逻辑
// 2. 当修改计算属性，会触发set函数
// 3. get方法必须返回一个值，这个值会返回给“属性名”
// 4. get方法的值会作为set方法的实参

vue指令 ​

TIP

定义：特殊的 html 标签属性（特点: v- 开头）

作用：每个 v- 开头的指令, 都有着自己独立的功能, 将来vue解析时, 会根据不同的指令提供不同的功能

1. v-bind ​

  • 语法：v-bind:属性名 = "值"
  • 简写：:属性名 = "值"
  • 作用：绑定标签的属性，使其值变成变量形式

 

2. v-on ​

  • 语法1：v-on:事件类型 = "执行语句"，简写：@事件类型 = "执行语句"
  • 语法2：v-on:事件类型 = "函数名(参数)"，简写：@事件类型 = "函数名(实参)"
  • 作用：给标签绑定事件
  • 其他：监听多个事件

 

3. v-show ​

  • 语法1：v-show="布尔值" （true显示, false隐藏）
  • 作用：控制盒子的显示/隐藏
  • 原理：实质是在控制元素的 css 样式，display: none;
  • 应用场景：如果是频繁的切换显示隐藏, 用v-show

 

4. v-if ​

  1. v-if：
  • 语法：v-if="布尔值" （true显示, false隐藏）
  • 作用：控制盒子的显示隐藏
  • 原理：实质是在动态的创建/删除元素节点
  • 应用场景：如果是不用频繁切换, 要么显示, 要么隐藏的情况, 适合于用 v-if
  1. v-else：
  • 语法：v-else="布尔值" （true显示, false隐藏）
  • 作用：控制盒子的显示隐藏
  1. v- else-if：
  • 语法：v-else-if="布尔值" （true显示, false隐藏）
  • 作用：控制盒子的显示隐藏

 

5. v-model ​

  • 语法：v-model = "值"
  • 作用：双向数据绑定，给表单元素使用，数据⇄视图
  • 注意：v-model 会忽略掉表单元素原本的value, checked等初始值
  • v-model修饰符：
  1. .number：把数据类型转为parseFloat类型；
  2. .trim：去除数据（变量）首尾空白字符；
  3. .lazy：表单失去焦点，才把值赋给数据（变量）

 

6. v-for ​

  1. 数组：
  • 语法：v-for = "(item，index) in 数组" :key = "item.id"
  • 作用：循环遍历生成列表

 

  1. 对象：
  • 语法：v-for = "(value，key) in 对象" :key = "key" （value是值，key是键）

 

  1. 数字：
  • 语法：v-for = "item in 数字" :key = "item"
  • 说明：item从1开始，包括数字

 

7. v-text ​

  • 语法：v-text="值"
  • 作用：更新元素的innerText
  • 注意：不会解析标签

 

8. v-html ​

  • 语法：v-html="值"
  • 作用：更新元素的innerHTML
  • 注意：会解析标签

 

动态控制样式 ​

TIP

1. 控制类 ​

  1. 语法1：:class="{类名: 布尔值}"
  • 如果键值对的值为true，那么就有这个类，否则没有这个类
  1. 语法2：:class="[类名1，类名2]"
  • 数组中所有的类，都会添加到盒子上
  1. 注意：
  • class 不会影响到原来的 class 属性
  • 类名如果带-，则只能用单引号引起来：:class="{'text-center': true}"

 

2. 控制行内样式 ​

  1. 语法1：:style="{样式名: 样式的值}"

  2. 语法2：:style="[{ 样式名: 样式的值 }, {样式名: 样式的值 }...]"

  3. 注意：

  • 样式名如果带横线，可以用小驼峰或者单引号

 

vue的渲染 ​

TIP

  1. 就地复用策略：数据改变，vue会尽可能复用旧的DOM，对比变化，只更新变化的部分

  2. 虚拟DOM：

  1. 作用：用来描述标签的信息的js对象
  2. 虚拟dom对比性能高
  3. 对比出变化的内容，减少dom操作，性能
  1. diff算法：
  1. 根节点：
  1. 如果类型变化，不再复用，删除旧dom，重新创建dom树；
  2. 如果类型不变，对比根节点属性，然后对比子节点
  1. 列表默认索引对比：
  1. 如果提供了key，按key进行对比，列表默认按索引对比；
  2. key主要是在列表顺序变化的时候，性能提升很明显
